## Assistant Guidelines

- Always run project tooling via `python dev.py <commande>`.
- Les commentaires/docstrings restent en anglais pour homogénéité.
- Préfère l’ASCII si possible; ajoute des commentaires seulement pour lever une ambiguïté.
- Tâche de limiter les dépendances au strict nécessaire; la librairie cible Django 4.2+ et 5.x.
- Quand tu ajoutes une nouvelle API publique, pense à l’illustrer rapidement dans le `README`.
- Les tests passent par `pytest`/`pytest-django` (commande `python dev.py test`).
